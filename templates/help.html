Instructions


Bingo 
Solve 5 questions in any row or column (not diagonal).
Duration: 1 hour
Challenges can be attempted in any order, should be submitted on our server, and will be approved or rejected within 2-3 minutes.
Each team will be given 3 systems, and team members can work on and submit problems simultaneously 
Internet access is provided but only for syntax related searches. Any team found violating this rule will be disqualified.
There will be no partial points for this round.
The first 4 teams to get BINGO will qualify to the finals.

Coding Problems 
Function headers have been given for Python, Java and C++, but if you would prefer a different language, you can ask for that language’s function header.
Complete the function and return the necessary value as asked in the question.
Taking input and/or printing the final output is not necessary.
Write the code in IDLE, VS Code, BlueJ, or any online compilers you wish to use. Submit your answers through our server.

Debug Problems 
Fix the errors in the existing code or rewrite the code to display the desired output. Questions have been given in Python, Java and C++.
Teams may rewrite any part/all of the code as per their wish, as long as it achieves the question's objective and passes the test cases.
Submit the entire corrected code.

Logic Problems 
Submit the answer required by the question. These questions will have a limited number of submissions- two per question. If two incorrect submissions are made, then the participants will not be able to solve that square, and must attempt a different row/column.

Gaming Challenges 
Complete the challenge and submit the code found at the end.
Gaming challenges can only be done on 1 system per team.
In case 4 teams do not get BINGO in 1 hour, the following criteria will be used to rank teams:
The first priority will be the total number of challenges solved correctly by a team, not necessarily in a row or column.
If 2 or more teams have solved the same number of questions, the number of questions solved in each category will be used, ranked in the following order: 
        Hard Coding > Gaming > Logic > Easy Coding > Debug
If there is still a tie, the time of submission of the last correctly solved problem will be taken.
